Variable Scope

072715

Scope tells us the places in a program that a specific variable can be accessed.

In other words, scope just tells us where a variable can be used and not used.

In Ruby, there are four types:

- local

- global

- instance

- class

Starting with local variables...

Local variables are only avaiable for use within the loop or method that it is declared in.

It's syntax is very simple:

local_variable = "I am local"

Global variables are available for use from anywhere in the program. It does not matter where it was declared.

It's syntax makes them easy to spot:

$global_variable = "The $ before the variable name declares it as a global variable"

Instance variables are available for use only for specific instances of an object.

In simpler terms, instance variables hold a value for one instance-- the next time, its value could be different.

Instance variables are written so:

@instance_variable = "only accessible within the class it is defined in, with its value varying depending on the instance"

Lastly, a class variable is a variable that is accessible by all instances of a class. Therefore, any change to the variable at any instance will cause it to change for everything else.

Class variables are written:

@@class_variable = "almost like a global variable, but only accessible within a class"
